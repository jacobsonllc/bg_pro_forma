## Ground Rules (working with Eli – a “bright beginner” contributor)

1. **Assume I’m a bright beginner.**  
   • Add docstrings, code comments, and plain-English reasoning in PR reviews.

2. **Everything lives in Git.**  
   • Even quick experiments → branch → commit → push.

3. **One-command runtime.**  
   • Each script/service must run headless with `make run`, `npm start`, etc.

4. **CI gates:** formatter + lint + unit tests must pass before merge.

5. **Explain commits like you would to a first-year CS student.**
